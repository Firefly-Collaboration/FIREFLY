  <div id="overview" style="padding:20px;background:linear-gradient(160deg,rgba(10,10,14,0.92),rgba(8,8,12,0.94));border:2px solid transparent;border-radius:14px;box-shadow:0 12px 30px rgba(0,0,0,0.4);background-image:linear-gradient(160deg,rgba(10,10,14,0.92),rgba(8,8,12,0.94)),linear-gradient(to left, #f7ba2b 0%, #ea5358 100%);background-origin:border-box;background-clip:padding-box,border-box;">
    <div style="font-size:22px;font-weight:800;margin-bottom:10px;color:#ffffff;">Overview</div>
    <p style="margin:0 0 10px 0;color:#f1f1f4;">FIREFLY is a chi-squared minimization fitting code designed for deriving the stellar population properties of stellar systems from spectroscopic data. Whether analysing the very lastest observed galaxy spectra, samples from huge surveys, or model spectra from simulations, FIREFLY provides a prior-free fitting approach.</p>
    <div style="display:grid;grid-template-columns:repeat(auto-fit,minmax(260px,1fr));gap:12px;margin-top:12px;">
      <div style="padding:12px;border:1px solid rgba(247,186,43,0.5);border-radius:12px;background:rgba(14,14,18,0.85);box-shadow:0 10px 20px rgba(0,0,0,0.35);">
        <div style="font-weight:800;color:#f7ba2b;margin-bottom:6px;">How It Works</div>
        <ul style="margin:0;padding-left:18px;color:#e6e6eb;line-height:1.4;">
          <li>FIREFLY fits combinations of single-burst stellar population (SSP) models to spectroscopic data following an iterative best-fitting process controlled by the Bayesian Information Criterion (BIC).</li>
          <li>No imposed priors: All solutions within a statistical cut are retained with their weights.</li>
          <li>No additive or multiplicative polynomials: Spectral shape is not artificially adjusted.</li>
          <li>No regularisation: Maximum fitting freedom to map intrinsic SED degeneracies, such as age, metallicity, dust reddening on stellar population properties.</li>
          <li>High-Pass Filter (HPF) dust treatment: Novel procedure for continuum rectification and dust attenuation. The returned attenuation array is then matched to known analytical approximations to return an E(B-V) value. This procedure allows for removal of large scale modes of the spectrum associated with dust and/or poor flux calibration.</li>
        </ul>
      </div>
      <div style="padding:12px;border:1px solid rgba(247,186,43,0.5);border-radius:12px;background:rgba(14,14,18,0.85);box-shadow:0 10px 20px rgba(0,0,0,0.35);">
        <div style="font-weight:800;color:#f7ba2b;margin-bottom:6px;">Performance</div>
        <ul style="margin:0;padding-left:18px;color:#e6e6eb;line-height:1.4;">
          <li>Comprehensive VACs produced from a variety of telescopes and different data sources.</li>
          <li>Extensivley tested on real galaxy spectra from the Sloan Digital Sky Survey (SDSS).</li>
          <li>Applied to spectra from SDSS-IV/MaNGA integral field spectroscopy to analyse millions of galaxies.</li>
          <li>Upgraded to derive the stellar population properties of galaxies from the latest Dark Energy Spectroscopic Instrument (DESI) observations.</li>
          <li>Tested on data from the DEEP2 survey and Milky Way globular clusters.</li>
          <li>Robust recovery down to S/N ~ 5 for moderately dusty systems.</li>
        </ul>
      </div>
    </div>
  </div>

  <div id="key-features" style="padding:20px;background:#0c0c11;border:2px solid transparent;border-radius:14px;box-shadow:0 12px 30px rgba(0,0,0,0.4);background-image:linear-gradient(#0c0c11,#0c0c11),linear-gradient(to left, #f7ba2b 0%, #ea5358 100%);background-origin:border-box;background-clip:padding-box,border-box;">
    <div style="font-size:22px;font-weight:800;margin-bottom:10px;color:#ffffff;">Key Features</div>
    <ul style="margin:0;padding-left:18px;color:#e6e6eb;line-height:1.5;">
      <li>Flexible Input: FIREFLY can be tailored to fit spectra from a vairety of different data formats, spectral resolutions and wavelength ranges.</li>
      <li>Comprehensive Output: Provides both light and mass weighted stellar population properties including age, metallicity, dust attenuation E(B-V), stellar mass and remnant mass partition (white dwarfs, neutron stars, black holes), star formation rates and histories, and SSP-specific component weights.</li>
      <li>Multiple Model Libraries: Support for MaStar (Maraston et al. 2020) and M11 (Maraston & Strömback 2011) stellar population models.</li>
      <li>Survey Integration: Dedicated piplines for DESI, SDSS, and MaNGA data sources.</li>
      <li>Emission Line Masking: Configurable masking settings for accurate continuum fitting.</li>
      <li>Multiple viable solutions: Using the Bayesian Information Criterion (BIC), FIREFLY retains several fits rather than a single best-fit, improving convergence with models and allowing for realistic error estimates on derived parameters.</li>
    </ul>
  </div>

  <div id="installation" style="padding:20px;background:linear-gradient(160deg,rgba(10,10,14,0.9),rgba(8,8,12,0.92));border:2px solid transparent;border-radius:14px;box-shadow:0 12px 30px rgba(0,0,0,0.4);background-image:linear-gradient(160deg,rgba(10,10,14,0.9),rgba(8,8,12,0.92)),linear-gradient(to left, #f7ba2b 0%, #ea5358 100%);background-origin:border-box;background-clip:padding-box,border-box;">
    <div style="font-size:22px;font-weight:800;margin-bottom:10px;color:#ffffff;">Installation</div>
    <div style="font-weight:700;color:#f7ba2b;margin-bottom:6px;">Requirements</div>
    <ul style="margin:0 0 12px 18px;padding:0;color:#e6e6eb;line-height:1.4;">
      <li>Python 3.6+</li>
      <li>Core packages: numpy, astropy and matplotlib.</li>
      <li>Additonal optional packages: argparse, multiprocessing, concurrent.futures, subprocess, fcnt and pandas.</li>
    </ul>
    <div style="font-weight:700;color:#f7ba2b;margin-bottom:6px;">Installation Steps</div>
    <ol style="margin:0 0 12px 18px;padding:0;color:#e6e6eb;line-height:1.5;">
      <li>Clone the repository:
        <pre style="background:#0f0f13;border:1px solid #f7ba2b;border-radius:10px;padding:10px;color:#f8f8fa;">git clone https://github.com/FIREFLY.git
cd firefly</pre>
      </li>
      <li>Install Python dependencies:
        <pre style="background:#0f0f13;border:1px solid #f7ba2b;border-radius:10px;padding:10px;color:#f8f8fa;">pip install -r requirements.txt</pre>
      </li>
      <li>Environment Variables (Optional):
        <div style="margin-top:6px;font-weight:700;color:#f7ba2b;">For bash/zsh (.bashrc or .bash_profile):</div>
        <pre style="background:#0f0f13;border:1px solid #f7ba2b;border-radius:10px;padding:10px;color:#f8f8fa;">export FF_DIR='/path/to/FIREFLY'
export PYTHONPATH="${FF_DIR}/firefly/Fitting_Engine:$PYTHONPATH"
export STELLARPOPMODELS_DIR="${FF_DIR}/firefly/Fitting_Engine/stellar_population_models"</pre>
        <div style="margin-top:6px;font-weight:700;color:#f7ba2b;">For csh/tcsh (.cshrc):</div>
        <pre style="background:#0f0f13;border:1px solid #f7ba2b;border-radius:10px;padding:10px;color:#f8f8fa;">setenv FF_DIR '/path/to/FIREFLY'
setenv PYTHONPATH ${FF_DIR}/firefly/Fitting_Engine:${PYTHONPATH}
setenv STELLARPOPMODELS_DIR ${FF_DIR}/firefly/Fitting_Engine/stellar_population_models</pre>
      </li>
    </ol>
    <div style="font-weight:700;color:#f7ba2b;">Stellar Population Models</div>
    <p style="margin:6px 0 0 0;color:#e6e6eb;">Stellar population model templates are included in the repository: M11 Models (Maraston & Strömback 2011): MILES, STELIB, ELODIE, MARCS libraries. MaStar Models: High-resolution empirical stellar library. IMF Options: Kroupa and Salpeter initial mass functions. Models are located in: firefly/Fitting_Engine/stellar_population_models/ and new models can be added to the module providing they are formatted for FIREFLY compatability.</p>
  </div>

    <ul style="margin:0 0 10px 18px;padding:0;color:#e6e6eb;line-height:1.4;">
      <li>MILES: Medium resolution (FWHM ~ 2.5Å)</li>
      <li>STELIB: Empirical stellar library</li>
      <li>ELODIE: High-resolution stellar library</li>
      <li>MARCS: Theoretical stellar atmospheres (Kroupa IMF only)</li>

  <div id="output-products" style="padding:20px;background:#0c0c11;border:2px solid transparent;border-radius:14px;box-shadow:0 12px 30px rgba(0,0,0,0.4);background-image:linear-gradient(#0c0c11,#0c0c11),linear-gradient(to left, #f7ba2b 0%, #ea5358 100%);background-origin:border-box;background-clip:padding-box,border-box;">
    <div style="font-size:22px;font-weight:800;margin-bottom:10px;color:#ffffff;">Output Products</div>
    <div style="font-weight:800;color:#f7ba2b;margin-bottom:6px;">Header Information</div>
    <ul style="margin:0 0 10px 18px;padding:0;color:#e6e6eb;line-height:1.4;">
      <li>stellar_mass: log(M*/M☉)</li>
      <li>age_lightW: Light-weighted log(age/yr)</li>
      <li>age_massW: Mass-weighted log(age/yr)</li>
      <li>metallicity_lightW: Light-weighted [Z/H]</li>
      <li>metallicity_massW: Light-weighted [Z/H]</li>
      <li>EBV: Dust attenuation E(B-V) in magnitudes</li>
      <li>ssp_number: Number of SSP components in best fit</li>
      <li>Individual SSP properties (log_age_ssp_X, metal_ssp_X, weightLight_ssp_X, weightMass_ssp_X)</li>
    </ul>
    <div style="font-weight:800;color:#f7ba2b;margin:6px 0;">Reading Output</div>
    <p style="margin:4px 0 6px 0;color:#e6e6eb;">Load FIREFLY output:</p>
    <pre style="background:#0f0f13;border:1px solid #f7ba2b;border-radius:10px;padding:10px;color:#f8f8fa;">from astropy.io import fits
import numpy as np

hdul = fits.open('output_file.fits')
data = hdul[1].data</pre>
    <p style="margin:8px 0 6px 0;color:#e6e6eb;">Extract best-fit parameters:</p>
    <pre style="background:#0f0f13;border:1px solid #f7ba2b;border-radius:10px;padding:10px;color:#f8f8fa;">wave  = data['wavelength']
flux  = data['original_data']
model = data['firefly_model']

stellar_mass = hdul[1].header['stellar_mass']
age_lightW = hdul[1].header['age_lightW']
metallicity = hdul[1].header['metallicity_lightW']
ebv = hdul[1].header['EBV']</pre>
    <p style="margin:8px 0 6px 0;color:#e6e6eb;">Display results:</p>
    <pre style="background:#0f0f13;border:1px solid #f7ba2b;border-radius:10px;padding:10px;color:#f8f8fa;">hdul.info()
hdul.close()

print('Age: ' + str(np.around(10**age_lightW, decimals=2)) + ' Gyr')
print('[Z/H]: ' + str(np.around(metallicity, decimals=2)) + ' dex')
print('log M/M☉: ' + str(np.around(stellar_mass, decimals=2)))
print('E(B-V): ' + str(np.around(ebv, decimals=2)) + ' mag')</pre>
    <p style="margin:8px 0 6px 0;color:#e6e6eb;">Plot spectrum and best-fit model:</p>
    <pre style="background:#0f0f13;border:1px solid #f7ba2b;border-radius:10px;padding:10px;color:#f8f8fa;">import matplotlib.pyplot as plt

plt.plot(wave, flux, label='Observed Spectrum')
plt.plot(wave, model, label='FIREFLY Model')
plt.xlabel('Wavelength')
plt.ylabel('Flux')
plt.legend()
plt.show()</pre>
